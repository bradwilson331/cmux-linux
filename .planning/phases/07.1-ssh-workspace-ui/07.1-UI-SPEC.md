# Phase 7.1 — UI Design Contract

> Visual and interaction contract for the SSH Workspace UI phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (GTK4 native) |
| Preset | not applicable |
| Component library | gtk4-rs 0.10.3 (v4_12 feature) |
| Icon library | none (text labels only) |
| Font | System default (inherited from GTK4 theme) |

**Note:** This project uses inline GTK CSS in `src/main.rs` (`APP_CSS` constant). All new styles for this phase append to that constant. No external stylesheets, no CSS preprocessor.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, attention dot margin |
| sm | 8px | Row vertical padding, compact element spacing |
| md | 16px | Row horizontal padding, dialog internal padding |
| lg | 24px | Dialog outer padding |
| xl | 32px | Not used this phase |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: none

**Source:** Existing `APP_CSS` uses 8px/16px row padding, 4px margins. This phase follows the same scale.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |

| Label (active) | 14px | 600 (semibold) | 1.5 |

## Component Inventory — SSH Connect Dialog

### Dialog Window

| Property | Value |
|----------|-------|
| Widget | `gtk4::Window` (not AlertDialog — no child widget support in gtk4 0.10) |
| Modal | true |
| Transient for | Main application window |
| Default width | 400px |
| Default height | auto (content-sized, ~80px) |
| Title | "Connect to SSH Host" |
| Background | #242424 (secondary surface, via CSS class `.ssh-dialog`) |
| Border radius | GTK4 default (follows system theme) |
| Padding (internal) | 16px (md) on all sides |

### Text Entry

| Property | Value |
|----------|-------|
| Widget | `gtk4::Entry` |
| Placeholder text | "user@host or SSH alias" |
| Font size | 14px |
| Full width | yes (fills dialog width minus padding) |
| Focus | Auto-grabbed on dialog open (`entry.grab_focus()`) |

### Autocomplete Dropdown

| Property | Value |
|----------|-------|
| Widget | `gtk4::EntryCompletion` (with `#[allow(deprecated)]`) |
| Model | `gtk4::ListStore` with single STRING column |
| Text column | 0 |
| Minimum key length | 1 character |
| Max visible items | 10 (GTK4 default — no override needed) |
| Source merge | Saved hosts (ssh_hosts.toml) + SSH config hosts (~/.ssh/config), deduplicated |

### Interaction Contract

| Action | Trigger | Result |
|--------|---------|--------|
| Open dialog | Ctrl+Shift+S | Dialog appears, entry focused, autocomplete loaded |
| Submit target | Enter key (in entry) | Dialog closes, SSH workspace created, sidebar shows new workspace with "reconnecting" state |
| Cancel | Escape key | Dialog closes, focus returns to active terminal pane |
| Select autocomplete | Arrow keys + Enter / Click | Entry text populated with selected host, then user presses Enter to connect |
| Focus restore | After dialog close (any path) | `focus_active_surface()` called to restore Ghostty terminal focus |

## Sidebar Integration

### New SSH Workspace Row (reuses existing patterns)

| Element | Visual |
|---------|--------|
| Workspace name | Auto-derived from hostname: `user@host.example.com` shows as "host.example.com" |
| Connection state label | 11px, below workspace name |
| Connection state: reconnecting | `#e8a444` text, shows "reconnecting..." |
| Connection state: connected | `#5b8dd9` text, shows "connected" |
| Connection state: disconnected | `#888888` text, shows "disconnected" |
| Active workspace | `#5b8dd9` background, `#ffffff` text (same as local workspaces) |

**Source:** Existing `APP_CSS` `.connection-state` classes. No new sidebar styles needed.

---

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Connect" (implicit via Enter key — no visible button) |
| Dialog title | "Connect to SSH Host" |
| Entry placeholder | "user@host or SSH alias" |
| Empty autocomplete | No dropdown shown (standard GTK4 behavior when no matches) |
| Connection error (terminal) | `[SSH: Connection refused -- user@host:22]` (existing Phase 7 pattern) |
| Connection error (sidebar) | "disconnected" state label in #888888 |
| Auth failure (terminal) | `[SSH: Authentication failed -- user@host]` (existing Phase 7 pattern) |
| Reconnecting state | "reconnecting..." in #e8a444 |
| Connected state | "connected" in #5b8dd9 |

**Destructive actions:** None in this phase. Dialog only creates workspaces; closing workspaces uses existing Ctrl+Shift+W flow.

## New CSS Rules

Append to `APP_CSS` in `src/main.rs`:

```css
/* Phase 7.1: SSH connect dialog (SSH-01) */
.ssh-dialog {
    background-color: #242424;
}
.ssh-dialog entry {
    font-size: 14px;
    padding: 8px 16px;
}
```

**Rationale:** Dialog background matches sidebar surface color. Entry padding uses sm vertical / md horizontal from spacing scale, matching the workspace row rhythm.
