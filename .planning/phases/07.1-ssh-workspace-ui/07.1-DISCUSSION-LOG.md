## Connection Dialog

| Option | Description | Selected |
|--------|-------------|----------|
| Simple text entry | Single input field for user@host. Like a browser address bar for SSH. | ✓ |
| Form with fields | Separate fields: username, hostname, port, identity file. | |
| Command palette style | Inline overlay like VS Code Ctrl+P. | |

**User's choice:** Simple text entry
**Notes:** Fast and minimal — matches the quick-access philosophy.

---

## Shortcut Binding

| Option | Description | Selected |
|--------|-------------|----------|
| Ctrl+Shift+S | S for SSH. Follows Ctrl+Shift pattern. Not currently bound. | ✓ |
| Ctrl+Shift+C | C for Connect. Conflicts with terminal copy expectations. | |
| Ctrl+Shift+Alt+N | Modifier on existing New Workspace shortcut. | |

**User's choice:** Ctrl+Shift+S
**Notes:** None.

---

## Saved Hosts

| Option | Description | Selected |
|--------|-------------|----------|
| Save in config.toml | Add [ssh.hosts] section to main config. | |
| No saved hosts | Always type fresh. | |
| Separate hosts file | Save to ~/.config/cmux/ssh_hosts.toml. | ✓ |

**User's choice:** Separate hosts file
**Notes:** Keeps SSH config separate from shortcut/display config.

## SSH Key / Auth Handling

| Option | Description | Selected |

## Workspace Naming

| Option | Description | Selected |
|--------|-------------|----------|
| Auto-name from host | Extract hostname, use as workspace name. | ✓ |
| Full target as name | Use full user@host string. | |
| Prompt for name | Ask user to name after connecting. | |

**User's choice:** Auto-name from host

## SSH Config Integration

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, for autocomplete | Parse Host entries for dropdown suggestions. | ✓ |
| No integration | Don't read ~/.ssh/config. | |
| Full integration | Parse identity files, ports, proxy settings too. | |

**User's choice:** Yes, for autocomplete only
