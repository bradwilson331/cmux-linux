  <action>
    Create `src/cli/format.rs` with human-readable output formatters per D-07 and D-08.

    **Color support (D-07):**
    - Define `use_color(color_flag: &str) -> bool`:
      - `"always"` -> true
      - `"never"` -> false
      - `"auto"` -> `std::io::stdout().is_terminal()` (Rust 1.70+ `IsTerminal` trait)
    - Helper `fn green(s: &str, color: bool) -> String` -> wraps in `\x1b[1;32m...\x1b[0m` if color
    - Helper `fn dim(s: &str, color: bool) -> String` -> wraps in `\x1b[2m...\x1b[0m` if color

    **List formatting (D-08):**
    `pub fn format_workspace_list(result: &Value, color: bool) -> String`:
    - Iterate `result["workspaces"]` array
    - For each: `{marker} {index}: {title} ({pane_count} panes)`
    - `marker` = `*` if `selected == true`, else ` `
    - Active line highlighted green if color=true
    - Example output: `* 1: Main (3 panes)`

    `pub fn format_surface_list(result: &Value, color: bool) -> String`:
    - Iterate `result["surfaces"]` array
    - For each: `{marker} {id_short} ({title})` where `id_short` = first 8 chars of UUID
    - `marker` = `*` if focused, else ` `

    `pub fn format_pane_list(result: &Value, color: bool) -> String`:
    - Similar to surface list format

    **Mutation formatting (D-09):**
    `pub fn format_mutation(command_name: &str, result: &Value) -> String`:
    - Extract `id` and `title`/`name` from result
    - Return `"Created workspace: {title} ({id})"` etc.

    **Generic formatting:**
    `pub fn format_response(command: &str, result: &Value, json_mode: bool, color: bool) -> String`:
    - If `json_mode`: return `serde_json::to_string_pretty(result)` (D-06: raw JSON result field)
    - Match on command name to pick the right formatter:
      - `"workspace.list"` -> `format_workspace_list`
      - `"surface.list"` -> `format_surface_list`
      - `"pane.list"` -> `format_pane_list`
      - `"system.ping"` -> just `"pong"` (or result["message"])
      - `"system.identify"` -> `"cmux {version} ({platform})"`
      - `"system.capabilities"` -> list methods
      - `"debug.layout"` -> pretty-print the layout tree
      - Default -> `serde_json::to_string_pretty(result)` (fallback for uncommon commands)

    **Wire into mod.rs:**
    Update the `run()` function to use `format::format_response()` instead of raw JSON printing:
    ```rust
    let output = format::format_response(method, &result, cli.json, use_color);
    print!("{}", output);
    ```
    For mutation commands, also print the D-09 success message before the formatted output (if not json mode).

    Add `pub mod format;` to `src/cli/mod.rs` (or however the module tree is structured from Plan 01).
