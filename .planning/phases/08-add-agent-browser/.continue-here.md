<current_state>
Adding browser keybindings and interactive preview pane to cmux-linux. Phase 08 plans (01-04) are complete. Working on post-phase feature work: making the browser preview pane functional with keyboard shortcuts, URL bar, viewport sizing, and mouse input forwarding.

The browser preview pane now renders CDP screencast frames (confirmed working — JPEG frames decoded to GDK Textures and displayed in gtk4::Picture). Mouse click and hover forwarding to headless Chrome is implemented but UNTESTED by the user yet.
</current_state>

<completed_work>

- Phase 08 plans 01-04: All complete with SUMMARYs and verification passed (4/4 must-haves)
- Cargo workspace: Added agent-browser/cli as workspace member so `cargo build` produces both binaries in target/debug/
- Keyboard shortcuts: Added `BrowserOpen` (Ctrl+Shift+B) and `BrowserClose` (Ctrl+Shift+Q) to config.rs + shortcuts.rs
- URL bar: Added gtk4::Entry to preview pane (create_preview_pane returns Box with Entry + Overlay + Picture)
- SplitNode::Preview: Changed container from Overlay to Box, added url_entry field
- Full daemon lifecycle: Ctrl+Shift+B now does stream_enable → launch → navigate(about:blank) → screencast_start → start_stream
- Frame rendering: WebSocket stream connected, JPEG frames decoded to GDK Textures, Picture widget updated (confirmed frame #1 received)
- Overlay label hidden: "No browser preview" label hidden when first frame arrives
- URL auto-scheme: Typing `www.google.com` auto-prepends `https://`
- Focus navigation: Preview panes reachable via Ctrl+Shift+Arrow, URL entry gets focus
- Close protection: Can't close last terminal pane when only Preview would survive (prevents Ghostty UAF crash)
- Viewport resize: Chrome viewport set to match pane dimensions (deferred idle + on each navigation)
- Mouse forwarding: GestureClick (mousePressed+mouseReleased) and EventControllerMotion (mouseMoved) attached to Picture, coordinates scaled from widget to viewport space
</completed_work>

<remaining_work>

- Mouse interaction: User reported they can see web pages but can't interact. Click/hover forwarding code is written but needs testing. May need coordinate scaling fixes.
- Keyboard forwarding: No keyboard input forwarding to Chrome yet (typing into web page form fields). Need to add EventControllerKey on Picture that forwards to input_keyboard daemon command.
- Page too small: Viewport resize is implemented but may still produce small renders if the initial about:blank screencast starts before the pane has its final size. May need to re-trigger screencast_start after viewport resize.
- Scroll forwarding: No mouse scroll wheel forwarding yet (for scrolling web pages). Need input_mouse with type=mouseWheel + deltaX/deltaY.
- Debug logging cleanup: Lots of diagnostic eprintln! calls that should be removed or gated behind a flag once everything works.
</remaining_work>

<decisions_made>

- Used Cargo workspace (members = [".", "agent-browser/cli"]) to build agent-browser alongside cmux-linux, so the binary is found automatically via which_agent_browser() checking alongside the cmux binary
- Changed SplitNode::Preview container from gtk4::Overlay to gtk4::Box to accommodate URL bar + overlay layout
- Mouse events forwarded via synchronous send_command to daemon (input_mouse action) — acceptable latency for click events
- Viewport sized to match Picture widget dimensions, not a fixed 1280x720
- URL entry activate signal uses state.borrow() (immutable) since send_command is &self
- Chose Ctrl+Shift+B (browser open) and Ctrl+Shift+Q (browser close) as defaults — configurable via config.toml
</decisions_made>

<blockers>
- Ghostty UAF on close: When closing the terminal pane next to a Preview (leaving Preview as sole root), ghostty_surface_free crashes. Worked around by preventing close of last terminal pane.
</blockers>
