<context>

The cancel race (fully understood):
1. wakeupCallback drainMailbox processes BOTH {.focus=false} AND {.focus=true} in one pass
2. {.focus=false}: cursor_h.cancel() submitted to xev loop; cursor_c = .active
3. {.focus=true}: guard `if cursor_c.state() != .active` → cursor_c IS .active → skip restart
4. drainMailbox finishes → renderCallback runs → drawFrame(false) → pushes redraw_surface
5. xev loop continues → cancel fires → cursorCancelCallback (OLD: ?*void, no restart)
6. cursor_c = .dead → timer permanently dead → cursor_blink_visible stuck at true

Fix makes step 5 aware of focus state:
5. cancel fires → cursorCancelCallback(Thread* self):
   if self.flags.focused (true) && cursor_c.state() != .active (true) → restart timer ✓
