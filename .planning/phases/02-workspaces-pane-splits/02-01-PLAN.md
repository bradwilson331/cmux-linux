<objective>
Migrate the single-surface Phase 1 globals in `callbacks.rs` and `surface.rs` to a multi-surface architecture that Phase 2's workspace/split code can build on. This is the prerequisite for everything else in Phase 2.

Purpose: Phase 1 stores exactly one GLArea (GL_AREA_FOR_RENDER thread-local) and one surface pointer (SURFACE_PTR AtomicUsize). Phase 2 creates N surfaces simultaneously. Without this refactor, wakeup_cb only renders one pane and close_surface_cb cannot identify which surface exited.

Output: Updated `callbacks.rs` with a GL_AREA_REGISTRY and SURFACE_REGISTRY; updated `create_surface()` signature that accepts per-pane parameters; `close_surface_cb` that does NOT call `process::exit`.
</objective>
