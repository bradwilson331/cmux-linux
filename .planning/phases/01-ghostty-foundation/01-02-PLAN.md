```markdown
# Ghostty Fork Changes (manaflow-ai/ghostty)

## GHOSTTY_PLATFORM_GTK4 — Linux GTK4 Embedded Variant

**Date:** 2026-03-23
**Phase:** 1 (Ghostty Foundation)
**Motivation:** Ghostty's embedded apprt has no Linux platform variant. This fork extension adds `GHOSTTY_PLATFORM_GTK4` to support embedding Ghostty surfaces in GTK4 GtkGLArea widgets from Rust.

### Files Changed

| File | Change |
|------|--------|
| `include/ghostty.h` | Added `GHOSTTY_PLATFORM_GTK4 = 3` to `ghostty_platform_e`; added `ghostty_platform_gtk4_s` struct (`void* gl_area`); added `gtk4` member to `ghostty_platform_u` |
| `src/apprt/embedded.zig` | Added `gtk4 = 3` to `PlatformTag`; added `GTK4` type and `gtk4` arm to `Platform` union; added `gtk4` to `Platform.C`; added `gtk4` arm to `Platform.init()` |
| `src/renderer/OpenGL.zig` | Added GTK4 dispatch in `surfaceInit` embedded arm: calls `prepareContext(null)` when platform is GTK4 (same as GTK apprt path). Added no-op GTK4 arm in `threadEnter`/`threadExit`. |

### Design Notes

- `ghostty_platform_gtk4_s.gl_area` uses `void*` (not `GtkWidget*`) to avoid requiring GTK4 headers in ghostty.h — consistent with macOS `void* nsview` pattern.
- The `GdkGLContext` is NOT passed in the struct; GTK4 manages it internally via `GtkGLArea`. The Rust layer calls `gl_area.make_current()` in the `realize` signal handler before any rendering.
- `prepareContext(null)` is the correct call for a global OpenGL context loaded by GTK4 (same as `apprt.gtk` uses).
- Thin fork diff: reuses `apprt/embedded.zig` infrastructure. No new Zig files. Per D-03.

### Conflict Notes for Upstream Rebase

- `ghostty_platform_e` enum will conflict if upstream adds new platform values at position 3.
- `Platform` union in embedded.zig will conflict on any change to Platform/PlatformTag.
- `OpenGL.zig` surfaceInit/threadEnter/threadExit switch blocks are conflict-prone during upstream renderer updates.
```
