3. Create `src/bin/generate.rs`. This binary imports the CLI definition from the cmux-linux crate and generates completions + man page. The key is using `CommandFactory` trait to get `clap::Command` from the `Cli` struct:

```rust
//! Standalone generator for shell completions and man page.
//! Usage: cargo run --bin cmux-generate
//! Outputs to packaging/completions/ and packaging/man/

use clap::CommandFactory;
use clap_complete::{generate_to, Shell};
use clap_mangen::Man;
use std::fs;
use std::path::Path;

// Import the Cli type from the library crate
use cmux_linux::cli::Cli;

fn main() -> std::io::Result<()> {
    let mut cmd = Cli::command();

    // Generate shell completions
    let comp_dir = Path::new("packaging/completions");
    fs::create_dir_all(comp_dir)?;

    for shell in [Shell::Bash, Shell::Zsh, Shell::Fish] {
        let path = generate_to(shell, &mut cmd, "cmux", comp_dir)?;
        eprintln!("Generated: {}", path.display());
    }

    // Generate man page
    let man_dir = Path::new("packaging/man");
    fs::create_dir_all(man_dir)?;

    let man = Man::new(cmd);
    let mut buf = Vec::new();
    man.render(&mut buf)?;
    fs::write(man_dir.join("cmux.1"), buf)?;
    eprintln!("Generated: packaging/man/cmux.1");

    Ok(())
}
```

IMPORTANT: This approach requires that `src/cli/mod.rs` is accessible from the library crate. Check if there is a `src/lib.rs` that exposes `pub mod cli`. If not, create one or adjust the import path.

The cmux-linux crate currently has two binary targets but may not have a `lib.rs`. If `src/lib.rs` does not exist:
- Create `src/lib.rs` with just: `pub mod cli;`
- This makes the cli module available as `cmux_linux::cli::Cli`

If the cli module has dependencies that prevent it from being in a library crate (e.g., it imports from other internal modules), then instead use an alternative approach: copy the path-based import. But the cli module appears self-contained (clap, serde_json, std only) so the lib.rs approach should work.
