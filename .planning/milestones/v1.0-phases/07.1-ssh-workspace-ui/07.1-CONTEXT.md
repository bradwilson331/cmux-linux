# Phase 7.1: SSH Workspace UI - Context

**Gathered:** 2026-03-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Provide a user-facing way to create SSH workspaces. Currently SSH workspace creation is only possible via raw socket JSON (`workspace.create` with `remote_target`). This phase adds a keyboard shortcut, GTK dialog, saved hosts, and SSH config integration — unblocking Phase 7 human UAT.

No changes to SSH connection logic, tunnel infrastructure, or I/O bridging. Just the UI entry point.

</domain>

<decisions>
## Implementation Decisions

### Connection Dialog
- **D-01:** Simple text entry dialog — single input field for `user@host` (or `user@host:port`). Like a browser address bar for SSH. No separate form fields.
- **D-02:** Dropdown autocomplete showing matching hosts from saved hosts file and `~/.ssh/config` as user types. Standard GTK4 completion pattern.

### Shortcut Binding
- **D-03:** Ctrl+Shift+S triggers the SSH connect dialog. Add `NewSshWorkspace` variant to `ShortcutAction` enum. Configurable via `config.toml` like all other shortcuts.

### Saved Hosts
- **D-04:** SSH hosts saved to separate file at `~/.config/cmux/ssh_hosts.toml` (not in main config.toml). Hosts auto-saved after successful connection.

### SSH Config Integration
- **D-05:** Parse `~/.ssh/config` Host entries for autocomplete suggestions in the dialog. The SSH command already uses the config transparently — this just surfaces aliases in the UI. No need to parse identity files, ports, or proxy settings.

### Error Feedback
- **D-06:** Sidebar shows connection state (connecting/failed/connected) via existing `ConnectionState` enum. Terminal pane shows error text like `[SSH: Connection refused — user@host:22]`. Reuses existing Phase 7 patterns.

### Authentication
- **D-07:** Rely on ssh-agent and system SSH command. No in-app password prompts. If auth fails, error appears in terminal pane. Users manage keys via standard SSH tooling.

### Workspace Naming
- **D-08:** Auto-name from hostname (extract from `user@host`, use just the host part). User can rename later with Ctrl+Shift+R (existing shortcut).

### Claude's Discretion
- GTK4 dialog styling details (padding, sizing, placeholder text)
- ssh_hosts.toml format (flat list vs sections for grouping)
- How many recent hosts to show in autocomplete dropdown
- Whether to deduplicate between ssh_hosts.toml and ~/.ssh/config entries

</decisions>

<specifics>
## Specific Ideas

- Dialog should feel fast — Ctrl+Shift+S, type target, Enter to connect. No extra clicks.
- Autocomplete merges saved hosts and SSH config hosts into one list
- Host auto-saved to `ssh_hosts.toml` after successful SSH handshake (not on dialog submit)

</specifics>

- SSH config editing from within cmux
- Host groups/folders in saved hosts

- Session resumption (tmux/screen on remote)
