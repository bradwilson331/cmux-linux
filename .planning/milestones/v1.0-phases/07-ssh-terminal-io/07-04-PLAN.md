<objective>
Fix the SSH terminal I/O pipeline by registering the initial remote pane in bridge.streams before the SSH lifecycle task starts.

Purpose: The UAT failure is caused by a missing pane registration. `create_remote_workspace` creates the pane and the IoWriteContext but never registers the pane_id in `bridge.streams`. When `run_proxy_routing` starts after SSH handshake, it iterates `bridge.streams` to open remote streams for each pane -- but the HashMap is empty, so `open_remote_stream` is never called. No stream means no proxy.write path for keystrokes and no proxy.stream.data events for output.

Output: Fixed `create_remote_workspace` that registers the initial pane, enabling the existing proxy routing to find and open a stream for it.
</objective>

       /// Register a pane with placeholder stream state (no stream_id yet).
       /// Called at workspace creation time so run_proxy_routing can find the pane
       /// and open a remote stream for it after SSH handshake.
       pub fn register_pane_placeholder(&self, pane_id: u64) {
           if let Ok(mut streams) = self.streams.lock() {
               streams.insert(pane_id, PaneStream {
                   stream_id: String::new(),
                   subscribed: false,
               });
           }
       }

       bridge.register_pane_placeholder(pane_id);
