# Phase 8 — UI Design Contract

> Visual and interaction contract for the agent-browser integration phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (GTK4 CSS — not a web frontend) |
| Preset | not applicable |
| Component library | GTK4 native widgets (gtk4-rs 0.10.x) |
| Icon library | none (no icons introduced this phase) |
| Font | System default (inherited from Ghostty/GTK4 theme) |

**Note:** cmux-linux is a Rust + GTK4 desktop application. All styling is done via GTK4 CSS loaded through `CssProvider`. No web design system (shadcn, Tailwind, etc.) applies.

|-------|-------|-------|

| sm | 8px | Compact element spacing, sidebar row padding (vertical) |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable (no user-facing button this phase; all commands via socket API or CLI) |
| Empty state heading | "No browser preview" |
| Empty state body | "Run `browser.open <url>` via the cmux socket or agent-browser CLI to start a browser session." |
| Error: Chrome not found | "Chrome not found. Run `agent-browser install` to download Chrome for Testing." |
| Error: daemon failed | "Browser daemon failed to start. Check agent-browser logs." |
| Error: stream disconnected | "Preview stream disconnected. Run `browser.stream.enable` to reconnect." |
| Error: frame decode failed | (silent — log to stderr only, keep last valid frame displayed) |
| Streaming indicator label | "LIVE" (shown on sidebar row when stream is active) |
| Destructive confirmation | `browser.close`: No confirmation required (socket command, not interactive UI). Daemon sends graceful shutdown to Chrome. |

**Notes:**
- All user-facing strings must use `String(localized:defaultValue:)` pattern per CLAUDE.md localization rules (adapted for Rust: use a localization-ready string constant pattern).
- Since Phase 8 is socket/CLI driven, most interaction happens through terminal panes. The preview pane itself has minimal interactive copy.

## Interaction States

### Preview Pane States

| State | Visual | Transition |
|-------|--------|------------|
| Empty (no session) | Dark background (`#1a1a1a`) + centered "No browser preview" label in `#888888` | `browser.open` triggers transition to Loading |
| Loading (daemon starting) | Dark background + "Starting browser..." label in `#e8a444` | Daemon ready triggers transition to Connected or Error |
| Connected (no stream) | Dark background + last screenshot if available, or empty state | `browser.stream.enable` triggers transition to Streaming |
| Streaming | JPEG frames displayed in `gtk4::Picture`, updated at ~5fps | `browser.stream.disable` or disconnect triggers transition to Connected or Error |
| Error | Dark background + error message label in `#cc4444` | Next successful command clears error state |

### Frame Update Contract

- Frames arrive as base64-encoded JPEG via WebSocket from agent-browser stream server
- Decode base64 to bytes in tokio task (off main thread)
- Create `gdk4::Texture::from_bytes()` on GTK main thread via `glib::idle_add_local_once`
- Call `picture.set_paintable(Some(&texture))` — single allocation per frame
- Target: 5fps, JPEG quality 60 (per RESEARCH.md recommendation)
- If frame decode fails: keep last valid frame, log error to stderr, do not show error overlay

### Pane Focus Behavior

- Preview pane is **not focusable** for keyboard input (no terminal surface)
- Preview pane can receive click focus for split navigation purposes
- When preview pane is "focused" in split tree, keyboard input falls through to no-op (no GLArea, no Ghostty surface)
- Active pane border (`1px solid #5b8dd9`) applies to Preview pane when it is the selected leaf in the split tree
