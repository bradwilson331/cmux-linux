## Interaction Contracts

### Bell Attention Flow

1. Terminal bell (`\a`) fires in any pane
2. Ghostty calls `action_cb` with `GHOSTTY_ACTION_RING_BELL` (tag 49)
3. `action_cb` identifies source pane via `SURFACE_REGISTRY` lookup
4. Sets `has_attention = true` on `SplitNode::Leaf`
5. Derives `workspace.has_attention = any(leaf.has_attention)`
6. Sidebar row shows attention dot (`.attention-dot` becomes visible)
7. If `!window.is_active()`: fires desktop notification (rate-limited to 1 per 5s per workspace)
8. When user switches to that workspace: `clear_workspace_attention()` sets all leaf `has_attention = false`, hides dot

### SSH Connection State Flow

1. Socket command `workspace.create { remote_target: "user@host" }` received
2. Sidebar row created immediately with "Reconnecting..." state
3. Tokio task deploys cmuxd-remote via scp, establishes SSH stdio connection
4. On success: state changes to "Connected", subtitle color changes to `#5b8dd9`
5. On network loss: state changes to "Disconnected" (`#888888`), begins exponential backoff
6. During reconnect attempts: state shows "Reconnecting..." (`#e8a444`)
7. On workspace close: SSH process is SIGTERM'd, sidebar row removed

### HiDPI Verification Points

1. Terminal surface: already handled by `notify::scale-factor` on GLArea (surface.rs:368)
2. Sidebar text: GTK4 scales GtkLabel automatically -- verify at 1x, 1.5x, 2x
3. Attention dot: CSS `px` units -- verify 8px dot renders correctly at all scales
4. Divider: existing CSS `min-width: 4px` -- verify at fractional scales
5. Connection state text: verify 11px renders legibly at 1x and above
