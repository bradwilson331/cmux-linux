<objective>
Fix desktop notifications: replace gio::Notification (which silently fails on Linux without
proper D-Bus registration, matching .desktop filename, and DBusActivatable=true) with
notify-rust crate that calls org.freedesktop.Notifications D-Bus interface directly.

Purpose: UAT test 2 failed — user heard bell but no desktop notification appeared.
Root cause: gio::Notification requires the .desktop file to be named after the app ID
(io.cmux.App.desktop), installed to XDG applications dir, and have DBusActivatable=true.
None of these are present. notify-rust bypasses all of this by talking to the notification
daemon directly.

Output: Working desktop notifications on any Linux desktop with a notification daemon.
</objective>

   ```rust
   fn send_bell_notification(app: &gtk4::Application, workspace_name: &str, workspace_index: usize) {
       use gtk4::gio;
       let notification = gio::Notification::new("Terminal Bell");
       notification.set_body(Some(&format!("{} - Terminal bell", workspace_name)));
       notification.set_priority(gio::NotificationPriority::Normal);
       let notif_id = format!("bell-{}", workspace_index);
       app.send_notification(Some(&notif_id), &notification);
   }
   ```

   Replace with:

   ```rust
   fn send_bell_notification(_app: &gtk4::Application, workspace_name: &str, _workspace_index: usize) {
       let summary = "Terminal Bell";
       let body = format!("{} - Terminal bell", workspace_name);
       std::thread::spawn(move || {
           if let Err(e) = notify_rust::Notification::new()
               .summary(summary)
               .body(&body)
               .icon("utilities-terminal")
               .timeout(notify_rust::Timeout::Milliseconds(5000))
               .show()
           {
               eprintln!("cmux: desktop notification failed: {e}");
           }
       });
   }
   ```

   The std::thread::spawn is needed because notify-rust's show() is synchronous and blocks
   while talking to D-Bus. The function is called from the GTK main thread (inside set_pane_attention),
   so blocking would freeze the UI. The _app and _workspace_index params are kept to avoid
   changing the call site signature.

3. Add `use notify_rust;` is NOT needed — the function uses the full path `notify_rust::Notification`.
   No other imports needed.

4. Do NOT change the call site in set_pane_attention — the function signature is unchanged.
