Add a new job `linux-build` to `.github/workflows/ci.yml` after the existing `web-typecheck` job and before the macOS `tests` job.

The job structure:

```yaml
  linux-build:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - name: Checkout
        uses: actions/checkout@34e114876b0b11c390a56381ad16ebd13914f8d5 # v4
        with:
          submodules: recursive

      - name: Install system deps
        run: |
          sudo apt-get update
          sudo apt-get install -y libgtk-4-dev libclang-dev libfontconfig1-dev \
            libfreetype6-dev libonig-dev libgl-dev

      - name: Install Zig
        uses: mlugg/setup-zig@v1
        with:
          version: 0.13.0

      - name: Build libghostty.a
        run: |
          cd ghostty
          zig build -Dapp-runtime=none -Doptimize=ReleaseFast -Dgtk-x11=true -Dgtk-wayland=true

      - name: Rust toolchain
        uses: dtolnay/rust-toolchain@stable
        with:
          components: clippy

      - name: Clippy
        run: cargo clippy --all-targets -- -D warnings

      - name: Build
        run: cargo build

      - name: Test
        run: cargo test --bin cmux-linux
```

**Important notes:**
- Use the SAME checkout action SHA as existing jobs (`actions/checkout@34e114876b0b11c390a56381ad16ebd13914f8d5`)
- The Zig version (0.13.0) should be verified against `ghostty/build.zig.zon` at implementation time. Check the file for `minimum_zig_version` or similar field. If a different version is required, use that instead.
- `submodules: recursive` is required for the ghostty submodule
- The `zig build` command matches `setup-linux.sh` flags: `-Dapp-runtime=none -Doptimize=ReleaseFast -Dgtk-x11=true -Dgtk-wayland=true` (DIST-04: both Wayland and X11 support)
- The system deps list includes the same packages as `setup-linux.sh` plus additional ones needed for linking (`libfontconfig1-dev`, `libfreetype6-dev`, `libonig-dev`, `libgl-dev`)
- Do NOT modify any existing macOS jobs

**3. Add `linux-appimage` job to `.github/workflows/release.yml`:**

Add after the existing `build-sign-notarize` job. This job runs independently (no `needs` dependency on the macOS job — they are parallel).

```yaml
  linux-appimage:
    runs-on: ubuntu-22.04
    timeout-minutes: 30
    steps:
      - name: Checkout
        uses: actions/checkout@34e114876b0b11c390a56381ad16ebd13914f8d5 # v4
        with:
          submodules: recursive

      - name: Install system deps
        run: |
          sudo apt-get update
          sudo apt-get install -y libgtk-4-dev libclang-dev libfontconfig1-dev \
            libfreetype6-dev libonig-dev libgl-dev

      - name: Install Zig
        uses: mlugg/setup-zig@v1
        with:
          version: 0.13.0

      - name: Build libghostty.a
        run: |
          cd ghostty
          zig build -Dapp-runtime=none -Doptimize=ReleaseFast -Dgtk-x11=true -Dgtk-wayland=true

      - name: Rust toolchain
        uses: dtolnay/rust-toolchain@stable

      - name: Build release binary
        run: cargo build --release

      - name: Download linuxdeploy
        run: |
          wget -q https://github.com/linuxdeploy/linuxdeploy/releases/download/continuous/linuxdeploy-x86_64.AppImage
          wget -q https://raw.githubusercontent.com/linuxdeploy/linuxdeploy-plugin-gtk/master/linuxdeploy-plugin-gtk.sh
          chmod +x linuxdeploy-x86_64.AppImage linuxdeploy-plugin-gtk.sh

      - name: Create AppImage
        run: |
          DEPLOY_GTK_VERSION=4 ./linuxdeploy-x86_64.AppImage \
            --appdir AppDir \
            --executable target/release/cmux-linux \
            --desktop-file resources/cmux.desktop \
            --icon-file resources/cmux.svg \
            --plugin gtk \
            --output appimage

      - name: Upload AppImage to release
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          APPIMAGE_FILE=$(ls cmux-*.AppImage 2>/dev/null | head -1)
          if [ -z "$APPIMAGE_FILE" ]; then
            echo "No AppImage file found" >&2
            exit 1
          fi
          TAG="${GITHUB_REF#refs/tags/}"
          gh release upload "$TAG" "$APPIMAGE_FILE" --clobber
```

**Important notes:**
- Pin to `ubuntu-22.04` (not `ubuntu-latest`) for AppImage reproducibility
- Use `DEPLOY_GTK_VERSION=4` env var for the GTK plugin (Pitfall 3)
- The Zig version should match what was determined in Task 1
- Use same checkout SHA as existing jobs
- Upload uses `gh release upload` with `--clobber` to handle re-runs
- The `linux-appimage` job has no `needs` — runs in parallel with macOS build
- Do NOT modify the existing `build-sign-notarize` job
