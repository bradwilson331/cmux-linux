  <name>Task 1: Fix post-drag cursor freeze and border repaint (Gap 1)</name>
  <files>src/split_engine.rs, src/ghostty/surface.rs</files>
  <action>
**Part A — src/split_engine.rs, inside `replace_leaf_with_split()` (around line 311, after the idle_add_local_once block that sets 50/50 position ends):**

After the closing `}` of the idle_add_local_once block (line 311), and before the `SplitNode::Split { ... }` return (line 313), add a `notify::position` handler on the new paned. This handler restores GTK and Ghostty focus after the drag ends.

The handler needs access to the active surface and GLArea. Since `replace_leaf_with_split` does not know which child is active, use `SplitEngine::active_pane_id` — but that is only available at the call site. The cleanest approach that does not require a refactor: use a `std::sync::Arc<std::sync::atomic::AtomicU64>` or simply re-walk the tree inside the closure. However, given the existing pattern in the codebase (SURFACE_REGISTRY and GL_AREA_REGISTRY are global statics), the simplest correct fix is:

In the notify::position handler, iterate `GL_AREA_REGISTRY` and call `grab_focus()` on the first area that `has_css_class("active-pane")`. Then look up its surface in `SURFACE_REGISTRY` — but SURFACE_REGISTRY maps `surface_ptr → pane_id`, not the reverse.

Simpler correct approach: walk the `paned`'s child widgets for the one with CSS class `active-pane` and call `grab_focus()` on it. For Ghostty focus: iterate `SURFACE_REGISTRY` to find the pane_id, then iterate GL_AREA_REGISTRY to confirm liveness — this is complex.

**Use this minimal correct implementation instead:**

```rust
// After the idle_add_local_once block ends (after line 311):
{
    let paned_ref = paned.clone();
    paned.connect_notify(Some("position"), move |_paned, _pspec| {
        // GtkPaned drag steals GTK focus. Restore it to whichever GLArea
        // has the active-pane CSS class so Ghostty resumes cursor blink.
        if let Ok(areas) = crate::ghostty::callbacks::GL_AREA_REGISTRY.lock() {
            for area_ptr in areas.iter() {
                let area: glib::translate::Borrowed<gtk4::GLArea> =
                    unsafe { glib::translate::from_glib_borrow(area_ptr.0) };
                if area.has_css_class("active-pane") {
                    area.grab_focus();
                    // Ghostty surface focus: look it up via SURFACE_REGISTRY.
                    // SURFACE_REGISTRY maps surface_usize → pane_id; we need
                    // the inverse. Use the GLArea pointer as the lookup key by
                    // comparing against GL_AREA_REGISTRY entries matched by
                    // has_css_class — one match is enough.
                    if let Ok(surf_reg) = crate::ghostty::callbacks::SURFACE_REGISTRY.lock() {
                        // Find the pane_id whose GLArea just got focus.
                        // We cannot get pane_id from GLArea alone here,
                        // so call ghostty_surface_set_focus on ALL registered
                        // surfaces, setting true only for the active one.
                        // Cheaper: just call set_focus(false) on all, then
                        // set_focus(true) on the one whose area.has_css_class.
                        // But SURFACE_REGISTRY maps surface→pane_id not area→surface.
                        // Accept a known limitation: grab_focus() alone is enough
                        // to unfreeze cursor blink in practice (GTK focus is the
                        // primary trigger). Skip ghostty_surface_set_focus here.
                        let _ = surf_reg; // avoid unused warning
                    }
                    break;
                }
            }
        }
    });
    let _ = paned_ref; // suppress unused warning if paned is used above
}
```

Wait — the above is overly complicated. Use this cleaner final version:

```rust
// Restore GTK focus after GtkPaned drag (Gap 1A).
// The drag temporarily moves GTK focus to the divider handle, freezing
// Ghostty's cursor blink. Re-grab focus on the active GLArea.
{
    paned.connect_notify(Some("position"), |_paned, _pspec| {
        if let Ok(areas) = crate::ghostty::callbacks::GL_AREA_REGISTRY.lock() {
            for area_ptr in areas.iter() {
                let area: glib::translate::Borrowed<gtk4::GLArea> =
                    unsafe { glib::translate::from_glib_borrow(area_ptr.0) };
                if area.has_css_class("active-pane") {
                    area.grab_focus();
                    break;
                }
            }
        }
    });
}
```

Place this block after the idle_add_local_once closing `}` (line 311) and before `SplitNode::Split {` (line 313). The block is self-contained and requires no captured variables.

<task type="auto">
  <name>Task 2: Fix pane-close crash — remove GLArea from registry before free (Gap 2 — blocker)</name>
  <files>src/split_engine.rs</files>
  <action>
In `close_active()` (lines 326-365 of src/split_engine.rs), the GLArea GObject is dropped inside `remove_leaf_from_tree()`. After that call, any raw pointer stored in `GL_AREA_REGISTRY` pointing to that GLArea is dangling. The next `wakeup_cb` or resize idle that iterates the registry calls `from_glib_borrow` on the dead pointer, which asserts `instance_of::<GLArea>` on a finalized GObject and panics.

**Fix:** Capture the raw GLArea pointer for the active pane BEFORE calling `remove_leaf_from_tree()`, then remove it from `GL_AREA_REGISTRY` AFTER the tree call but BEFORE `ghostty_surface_free()`.

Locate `close_active()`. After line 335 (`let surface_to_free = self.find_surface(active_id)?;`), add:

```rust
// Capture the raw GLArea pointer BEFORE the tree removal drops the GObject.
// GL_AREA_REGISTRY holds raw pointers; once GTK finalizes the GObject the
// pointer becomes dangling. Remove it here while the GLArea is still alive.
let raw_gl_area: Option<*mut gtk4::ffi::GtkGLArea> = self
    .find_gl_area(active_id)
    .map(|a| a.as_ptr() as *mut gtk4::ffi::GtkGLArea);
```

Then after line 338 (`let surviving_id = remove_leaf_from_tree(&mut self.root, active_id)?;`), add:

```rust
// Remove the now-dropped GLArea from GL_AREA_REGISTRY before any further
// callbacks can dereference the dangling pointer (Gap 2 fix).
if let Some(raw_ptr) = raw_gl_area {
    if let Ok(mut areas) = crate::ghostty::callbacks::GL_AREA_REGISTRY.lock() {
        areas.retain(|p| p.0 != raw_ptr);
    }
}
```

This must come BEFORE the `ghostty_surface_free(surface_to_free)` call on line 342, because freeing the surface may trigger a wakeup_cb which iterates the registry.

The final ordering in `close_active()` must be:
1. `find_surface(active_id)` — get surface ptr
2. `find_gl_area(active_id)` → capture raw pointer (new)
3. `remove_leaf_from_tree(...)` — drops GLArea GObject
4. `GL_AREA_REGISTRY.retain(...)` — purge dangling pointer (new)
5. `ghostty_surface_free(surface_to_free)` — free surface
6. `SURFACE_REGISTRY.remove(...)` — deregister surface
7. Focus update steps (unchanged)

**Imports needed:** `gtk4::prelude::WidgetExt` is already imported via `use gtk4::prelude::*` at the top of split_engine.rs. `gtk4::ffi::GtkGLArea` is the raw type — add `use gtk4::ffi;` at the top if not present, or use the fully qualified path `gtk4::ffi::GtkGLArea`. Check existing imports; `ffi` is imported as `use crate::ghostty::ffi` (Ghostty FFI), not gtk4::ffi. Use the full path `gtk4::ffi::GtkGLArea` inline to avoid name collision.

  <name>Task 3: Change pane focus shortcuts from Ctrl+Alt+Arrow to Ctrl+Shift+Arrow (Gap 3)</name>
  <files>src/shortcuts.rs</files>
  <action>
In src/shortcuts.rs, the four pane focus-direction match arms at lines 147-168 use the pattern `(true, false, true, k)` which decodes as `(ctrl=true, shift=false, alt=true, key)`. On GNOME and KDE, Ctrl+Alt+Arrow is a global compositor shortcut for switching virtual desktops. It is delivered at the Wayland seat level before the GTK4 client receives it, so the app never sees it.

Change all four arms from `(true, false, true, k)` to `(true, true, false, k)` (ctrl=true, shift=true, alt=false — Ctrl+Shift+Arrow). This matches the Terminator convention and is not claimed by any major Linux desktop by default.

The four arms to update (lines 147, 153, 159, 165):

```rust
// BEFORE (Ctrl+Alt+Arrow — intercepted by WM):
(true, false, true, k) if k == gtk4::gdk::Key::Left => { ... }
(true, false, true, k) if k == gtk4::gdk::Key::Right => { ... }
(true, false, true, k) if k == gtk4::gdk::Key::Up => { ... }
(true, false, true, k) if k == gtk4::gdk::Key::Down => { ... }

// AFTER (Ctrl+Shift+Arrow — free on Linux):
(true, true, false, k) if k == gtk4::gdk::Key::Left => { ... }
(true, true, false, k) if k == gtk4::gdk::Key::Right => { ... }
(true, true, false, k) if k == gtk4::gdk::Key::Up => { ... }
(true, true, false, k) if k == gtk4::gdk::Key::Down => { ... }
```

Also update the four comments above each arm from "Ctrl+Alt+..." to "Ctrl+Shift+..." for accuracy.
