## Authentication & Identity

**Auth Provider:**
- Custom (local socket-based) - No external identity provider
  - Implementation: `SocketControlSettings.swift`, `TerminalController.swift`
  - Socket authentication: Optional password-based (local machine only)
  - Command: `auth <password>` required when socket access control enabled
  - Message: "Authentication required. Send auth <password> first."

**Socket Security:**
- Unix domain socket (`SocketControlSettings.stableDefaultSocketPath`)
- Optional password authentication for socket commands
- Access mode validation per command
- No external OAuth/SSO integration

**Privacy & Telemetry Control:**
- Opt-out telemetry: User setting `sendAnonymousTelemetry`
- Default: Enabled (true)
- Freezes at launch: `TelemetrySettings.enabledForCurrentLaunch`
- User can disable in Settings view
- Environment override: `CMUX_POSTHOG_ENABLE` (DEBUG builds only)

## Monitoring & Observability

**Error Tracking:**
- Sentry (see Sentry entry above)
  - `sentryBreadcrumb()` - Add context breadcrumbs
  - `sentryCaptureWarning()` - Warning-level errors
  - `sentryCaptureError()` - Error-level errors
  - All calls guarded by telemetry settings

**Logs:**
- Local file system only
  - Debug event log: `/tmp/cmux-debug.log` (untagged) or `/tmp/cmux-debug-<tag>.log` (tagged)
  - File written by `DebugEventLog` (DEBUG builds only)
  - Ring buffer: 500 entries max
  - Events logged: keys, mouse, focus, splits, tabs, bonsplit operations

**Telemetry:**
- PostHog (see PostHog entry above) for analytics
- Anonymous distinct ID auto-generated by SDK
- Super properties include platform and version

## CI/CD & Deployment

**Hosting:**
- GitHub - Code hosting
- Sparkle - Auto-update distribution
  - Feed URL: `https://github.com/manaflow-ai/cmux/releases/latest/download/appcast.xml`
  - Automatic update checks every 86400 seconds (24 hours)
  - Manual update checking enabled: `SUEnableAutomaticChecks = true`
  - Automatic updating disabled: `SUAutomaticallyUpdate = false`

**CI Pipeline:**
- GitHub Actions (primary)
  - Workflows: `ci.yml`, `nightly.yml`, `release.yml`, `test-e2e.yml`
  - Test runners: WarpBuild macOS 15 ARM64
  - Release process: Tagged releases trigger `release.yml`
  - E2E tests: Separate `test-e2e.yml` workflow

**Code Signing & Release:**
- GitHub Actions secrets (hardcoded DSN/keys, not in repo):
  - `APPLE_CERTIFICATE_BASE64` - Code signing certificate
  - `APPLE_CERTIFICATE_PASSWORD` - Certificate password
  - `APPLE_SIGNING_IDENTITY` - Developer identity
  - `APPLE_ID`, `APPLE_APP_SPECIFIC_PASSWORD` - Apple ID credentials
  - `APPLE_TEAM_ID` - Team identifier
  - `SPARKLE_PUBLIC_KEY` - Auto-update verification key

**Release Artifacts:**
- DMG package: `cmux-macos.dmg` (attached to GitHub releases)
- AppCast XML: `appcast.xml` (for Sparkle updates)
- Version management via `MARKETING_VERSION` and `CURRENT_PROJECT_VERSION` in Xcode

## Environment Configuration

**Required env vars for development:**
- `CMUX_SOCKET` - Custom socket path (default: `~/.cmux/sockets/cmux.sock`)
- `DEVELOPER_DIR` - Xcode developer tools path (for CI)

**Optional env vars:**
- `CMUX_DISABLE_SESSION_RESTORE` - Disable restoring previous sessions (set to "1" to disable)
- `CMUX_RESTORE_SCROLLBACK_FILE` - Path to scrollback history file
- `CMUX_FEEDBACK_API_URL` - Override feedback endpoint
- `CMUX_POSTHOG_ENABLE` - Force PostHog in DEBUG builds (set to "1")
- `CMUX_POSTHOG_DEBUG` - PostHog SDK debug mode (set to "1")

**Secrets location:**
- GitHub Actions secrets (production DSN, Apple credentials)
- Not checked into repo (follows best practices)
- Sentry DSN: Embedded in source as hardcoded string (production only)
- PostHog API key: Embedded in source (public key, non-sensitive)

## Webhooks & Callbacks

**Incoming:**
- AppleScript callbacks via `cmux.sdef` scripting dictionary
  - Commands: `openTab`, `openWindow`
  - Handlers: `AppleScriptSupport.swift`
- Custom URL schemes: `http://`, `https://` handlers
  - Configured in `Info.plist` > `CFBundleURLTypes`
  - Used for terminal link clicks and `open` command interception

**Outgoing:**
- Feedback API: POST to `CMUX_FEEDBACK_API_URL` (custom endpoint)
- Sparkle auto-update: GET request to appcast XML feed
  - URL: `https://github.com/manaflow-ai/cmux/releases/latest/download/appcast.xml`
- Sentry error reporting: POST to Sentry ingest endpoint
- PostHog analytics: POST to PostHog ingestion endpoint

**Browser API Commands (WebSocket-like via socket):**
- CLI socket-based command protocol (not HTTP)
- Commands: `browser.*`, `window.*`, `workspace.*`, `surface.*`, `pane.*`, `tab.*`
- Implementation: `TerminalController.swift` socket command handler
- Response format: Text-based protocol (OK, ERROR, JSON payloads)

## Network Configuration

**DNS & Hosts:**
- Loopback exception: `cmux-loopback.localtest.me`
  - Allows insecure HTTP loads for local development
  - Configured in `Info.plist` > `NSAppTransportSecurity` > `NSExceptionDomains`

**App Transport Security (ATS):**
- General rule: HTTPS required
- Exception: Arbitrary loads allowed in web content (WKWebView)
  - `NSAllowsArbitraryLoadsInWebContent = true`
- Exception domain: `cmux-loopback.localtest.me` (insecure HTTP allowed)

**Port Allocation:**
- Configurable port base: `cmuxPortBase` (default: 9100)
- Configurable port range: `cmuxPortRange` (default: 10)
- Port scanning: `PortScanner.swift` for browser port detection
